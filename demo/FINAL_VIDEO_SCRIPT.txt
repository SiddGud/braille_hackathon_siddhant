Getting this to work flawlessly required an intense, custom image processing pipeline. 

**[SCREEN: Point cursor to Stage 2 and Stage 3]**

When the noisy image reaches my backend, I run it through an OpenCV Bilateral Filter to smooth the paper texture while keeping the Braille dots perfectly sharp. I then feed this cleaned image into my YOLOv8 foundation model using Test-Time Augmentation to catch hidden dots at multiple zoom levels.

**[SCREEN: Point cursor to Stage 4]**

Finally, I spatially sort the dots and run them through a custom Python autocorrection layer—a spatial spellchecker that fixes minor physical errors to guarantee perfect output.

**[SCREEN: Switch to the GitHub README page, showing the "Engineering Journey & Pipeline" section.]**

Building this as a solo developer was tough. Let me share the two biggest challenges I faced.

First, the "Heavy Model" hallucination. Even though my massive models hit 93% accuracy, they hallucinated on physical shadows. I realized the secret wasn't a massive model—it was using a lighter foundation model paired with intense OpenCV preprocessing.

Second, the flip augmentation disaster. My model kept confusing the letter "r" and "w". I realized my script was horizontally flipping images to increase dataset size. But Braille is chiral! Mirroring an "r" literally turns it into a "w". Disabling flips saved my entire project.
